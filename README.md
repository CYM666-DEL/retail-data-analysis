# 🛒 零售业务数据分析与洞察报告 (2010-2011)

**一个使用纯SQL与Excel完成的端到端数据分析项目**，完整展示了从数据查询、清洗、分析到生成商业建议的全流程。本项目旨在模拟商业分析师的核心工作：**用数据回答关键业务问题，并驱动决策**。

## 📊 项目核心价值

- **技术纯粹性**：仅使用 **SQL** 和 **Excel** 两大最通用、最受商业环境认可的工具，证明解决复杂分析问题的能力。
- **完整分析闭环**：从原始数据到可执行的商业建议，形成 **“提问 → 分析 → 可视化 → 建议”** 的完整逻辑链。
- **可复现与可验证**：所有SQL脚本和Excel处理步骤清晰、可追溯，结论经得起检验。
- **业务导向**：结论直接服务于业务策略，而非单纯的数据展示。

## 🔍 核心商业发现

| 分析维度 | 核心数据结论 | 对业务意味着什么？ |
| :--- | :--- | :--- |
| **业务模式诊断** | 营收呈现 **“批发为主，零售为辅”** 的混合模式。其中，**头部大宗批发客户单笔消费可达 £168,469**。 | 公司拥有稳定的大客户基本盘，但需关注客户集中度风险，并思考如何提升零售端利润。 |
| **增长与健康度** | 销售额年增长率 **338%**，客户数增长 **208%**，同时保持 **1.85%** 的极低退货率。 | 业务处于**高速健康增长期**，市场与运营能力得到双重验证，是扩张的良好时机。 |
| **客户结构洞察** | 客户梯队完整：从**高净值批发客户**到**年交易达14次的忠诚零售客户**并存。 | 客户生态健康，为实施 **“客户分层精细化运营”**（如会员体系、差异化服务）提供了完美基础。 |

## 🛠️ 技术流程与工具

**核心流程：**
1.  **数据获取与清洗**：使用 **SQL** 编写复杂查询，从数据库中精准提取所需数据，并进行初步关联与筛选。
2.  **深度分析与建模**：使用 **Excel** 进行数据透视、公式计算（如`SUMIFS`, `VLOOKUP`）。
3.  **可视化与呈现**：使用 **Power BI** 构建交互式管理看板，直观呈现关键指标趋势与分布。
4.  **故事化报告**：将分析过程与结论整合成结构化报告，直接服务于商业决策。

**主要工具栈：** `SQL` | `Excel` (数据透视表、函数) | `Power BI`
